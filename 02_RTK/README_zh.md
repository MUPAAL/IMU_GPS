# 02_RTK — 输出字段说明

WebSocket 输出帧（`ws://localhost:8776`），约 5 Hz 广播。

## 字段说明

| 字段          | 单位    | 说明                                                                   |
|---------------|---------|------------------------------------------------------------------------|
| `type`        | —       | 固定为 `"rtk_frame"`                                                   |
| `version`     | —       | 协议版本，当前为 `1`                                                    |
| `lat`         | 度      | 纬度，WGS-84 十进制度（+北/-南）                                          |
| `lon`         | 度      | 经度，WGS-84 十进制度（+东/-西）                                          |
| `alt`         | m       | 平均海平面以上高度                                                     |
| `fix_quality` | —       | 定位类型（见下表）                                                       |
| `num_sats`    | 颗      | 当前使用的卫星数量                                                     |
| `hdop`        | —       | 水平精度因子。越小越好（< 1.0 优秀，> 2.0 较差）                           |
| `speed_knots` | 节      | 来自 RMC 句子的地速                                                    |
| `track_deg`   | 度      | 来自 RMC 句子的地面航向（0 = 正北，90 = 正东）                            |
| `rtk_ts`      | Unix 秒 | GPS 接收机的时间戳（来自 GGA 句子，真实时钟）                             |
| `server_ts`   | Unix 秒 | bridge 序列化此帧时的主机时间戳                                        |
| `source`      | —       | `"rtk"` = 真实定位，`"default"` = 尚未定位（显示 config.py 里的默认坐标） |

## `fix_quality` 取值

| 值 | 含义                          |
|----|-------------------------------|
| 0  | 无定位                        |
| 1  | 普通 GPS                      |
| 2  | DGPS 差分                     |
| 4  | RTK 固定解，精度最高（约 1 cm）  |
| 5  | RTK 浮动解，精度较好（约 10 cm） |

## 注意事项

- **`source: "default"`** 表示还没有收到任何 GPS 定位帧，`lat`/`lon` 显示的是 `config.py` 里定义的默认坐标，不是真实位置。
- **`rtk_ts` vs `server_ts`：** `rtk_ts` 来自 GPS 接收机内部时钟（精确）；`server_ts` 是主机广播该帧时的时间，两者之差是串口读取和解析的延迟。
- `speed_knots` 和 `track_deg` 来自 RMC 句子，当 RMC 状态为无效时为 `null`。
