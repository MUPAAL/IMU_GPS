# 01_IMU — 输出字段说明

WebSocket 输出帧（`ws://localhost:8766`），约 50 Hz 广播。

## 顶层字段

| 字段      | 单位 | 说明                                        |
|-----------|------|---------------------------------------------|
| `type`    | —    | 固定为 `"imu_frame"`                        |
| `version` | —    | 协议版本，当前为 `1`                         |
| `hz`      | Hz   | bridge 实测的当前帧率                       |
| `ts`      | ms   | ESP32 时间戳（开机后毫秒数，不是真实时钟）     |
| `cal`     | 0–3  | BNO085 校准状态。`3` = 完全校准，`0` = 未校准 |
| `steps`   | 次   | 计步器数值（行人功能，机器人场景无意义）       |

## `rot` — 旋转向量（带磁力计，参考磁北）

BNO085 旋转向量传感器输出的四元数，融合了磁力计，因此方向参考磁北。

| 字段           | 说明                       |
|----------------|----------------------------|
| `qi` `qj` `qk` | 四元数向量部分（i、j、k 分量） |
| `qr`           | 四元数标量部分             |

## `euler` — 欧拉角

由 `rot` 四元数推导，ZYX 旋转顺序。

| 字段               | 单位 | 说明                                         |
|--------------------|------|----------------------------------------------|
| `roll`             | 度   | 绕 X 轴旋转（左右倾斜）                        |
| `pitch`            | 度   | 绕 Y 轴旋转（前后倾斜）                        |
| `yaw`              | 度   | 绕 Z 轴旋转（0 = 传感器参考方向，不一定是正北） |
| `north_offset_deg` | 度   | 北偏校正量（通过浏览器 UI 设置）               |

## `heading` — 磁北航向

由 `rot` 四元数推导，并加入 `north_offset_deg` 校正。这是导航用的主航向输出。

| 字段  | 单位 | 说明                                                   |
|-------|------|--------------------------------------------------------|
| `raw` | 度   | 校正前的原始航向                                       |
| `deg` | 度   | 校正后的航向（0 = 正北，90 = 正东，180 = 正南，270 = 正西） |
| `dir` | —    | 16 方位罗盘标签（N、NNE、NE、ENE、E……）                      |

## `game_rot` — 游戏旋转向量（不用磁力计）

BNO085 游戏旋转向量传感器输出的四元数，**不使用磁力计**。Yaw 会随时间漂移，但不受磁场干扰，适合短时间内的姿态跟踪。

| 字段           | 说明           |
|----------------|----------------|
| `qi` `qj` `qk` | 四元数向量部分 |
| `qr`           | 四元数标量部分 |

## `accel` — 加速度计

包含重力的原始加速度。单位：m/s²。

| 字段        | 说明       |
|-------------|------------|
| `x` `y` `z` | 各轴加速度 |

## `lin_accel` — 线性加速度

去除重力后的加速度（传感器融合计算）。单位：m/s²。静止时接近零。

## `gravity` — 重力向量

仅包含重力分量（传感器融合计算）。单位：m/s²。可用于独立于运动状态的倾斜估计。

## `gyro` — 陀螺仪

角速度。单位：rad/s。静止时接近零。

| 字段        | 说明         |
|-------------|--------------|
| `x` `y` `z` | 各轴旋转速率 |

## `mag` — 磁力计

原始磁场强度。单位：µT。BNO085 内部用于航向融合；附近有铁质材料或电子设备时会受干扰。

| 字段        | 说明         |
|-------------|--------------|
| `x` `y` `z` | 各轴磁场强度 |

## 注意事项

- **导航用哪个航向：** 用 `heading.deg`，它经过了磁北校正，是最终输出。
- **`cal: 0` 表示航向不可靠。** 拿着传感器做"8"字形挥动可以完成磁力计校准。
- **`ts` 不是真实时钟。** ESP32 重启后会归零。如需绝对时间，在 bridge 侧用 `time.time()`。
